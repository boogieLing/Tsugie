# Node 0:1 MVP 详情页与状态演示归档 v2

## 1. 本阶段需求摘要
- 在地图首页保持“点位 -> quickCard -> 上划全屏详情页”的主链路。
- 详情页补齐核心信息：地点名、距离、开放时间、状态时间、活动图、地图 focus 区、介绍、热度、惊喜度。
- 引入状态进度组件：
  - 进行中：荧光渐变进度条。
  - 未开始：半透明虚拟时间轴。
- quickCard 也接入同一进度组件（精简版）。
- 支持 MVP 演示：不同地点固定不同状态，便于联调与演示。

## 2. 已实现功能点
1. 交互链路
- 点击地图点位打开 quickCard。
- quickCard 上划展开全屏详情页。
- 详情页下滑关闭并回到 quickCard。
- 详情页内“フォーカス”可回到地图对应地点。

2. 状态与时间
- 进行中/未开始/已结束状态可识别并驱动 UI。
- 倒计时按秒实时刷新。
- quickMeta 增加日期语义：
  - 开始日是当天时显示 `本日開催！まもなく！`。
  - 否则显示 `YYYY-MM-DD`。

3. 进度组件
- quickCard 与详情页复用统一结构：`track + fill + endpoint(へ)`。
- endpoint 设计为 `へ` 小胶囊。
- quick 的进行中 fill 增加动态朦胧发光层（发光色与 fill 颜色不同，按进度实时混色）。

4. MVP 多状态演示配置（测试态）
- 隅田川：未开始（upcoming）。
- 浅草：进行中约 1/3（ongoing）。
- 目黑川：已结束（ended）。
- 神宫外苑：跨日未开始（upcoming，30h 后）。

## 3. 设计注意点（当前结论）
- 信息层级：quickCard 只保留决策最短路径信息，详情页承载完整语义。
- 组件一致性：进度组件跨 quick/detail 统一，避免两套视觉语义。
- 状态可读性：状态颜色和结构明确，但避免过重底色；轨道背景保持轻量半透明。
- 动效克制：发光只作用于“进行中”的 quick 进度条，避免全局噪声。
- 边缘处理：进度终端与上方元素需做左右锚点，避免超出屏幕。
