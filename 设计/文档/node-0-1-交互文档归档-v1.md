# Node 0:1 交互文档归档 v1

## 1. 首次进入流程
- 进入页面。
- 显示地图与点位。
- 右上角显示随机颜文字入口。
- 底部默认显示附近轮播。
- 约 2.2 秒后自动弹出推荐点位的最速攻略卡片。
- 弹卡后附近轮播自动隐藏。

## 2. 地图点位交互
- 用户点击某个点位。
- 被点击点位进入高亮态（发光 + 律动）。
- 地图聚焦该点位。
- 底部最速攻略卡片出现并更新：标题、meta、提示语。
- 附近轮播隐藏。

## 3. quickCard 与轮播互斥交互
- quickCard 显示：轮播隐藏。
- quickCard 关闭：轮播显示并刷新候选。
- 轮播点选：触发与地图点位点击一致的弹卡逻辑。

## 4. 附近轮播交互
- 轮播支持左右滑动。
- 每张卡片两行信息：
- 第一行：地点名（左）+ 距离（右）。
- 第二行：剩余时间（左）+ 开始时间（右）。
- 点击卡片直接打开对应最速攻略。

## 5. 当前位置交互
- 首屏定位点固定在测试坐标（隅田川/天空树附近）。
- 加载时定位点视觉位于屏幕中央偏下。
- 点击右侧定位按钮：快速回到定位点。
- 在降级模式下，按钮点击触发覆盖层定位点脉冲反馈。

## 6. 地图模式与异常交互
- JS API 可用：交互地图模式。
- JS API 不可用：自动降级 Embed/覆盖层模式。
- 降级时仍可点击点位并打开 quickCard。
- 异常提示通过 `mapHint` 显示（文案为日文）。

## 7. 动效原则（当前）
- quickCard：仅底部滑入/滑出，无透明度过渡。
- 点位高亮：强调当前选择，弱化非选中点。
- 定位点：轻脉冲提示当前位置，不干扰主决策流程。
