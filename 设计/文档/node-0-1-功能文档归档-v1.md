# Node 0:1 功能文档归档 v1

## 1. 功能目标
- 首屏打开即地图，先给“去哪儿”的决策入口。
- 通过点位与最速攻略卡片，帮助用户快速判断下一步。
- 在地图放大场景下，持续提示“画面外一点点”的可选地点。

## 2. 功能清单（当前可用）
- 地图首页：全屏地图展示。
- 点位展示：地图内展示可点击 `へ` 点位。
- 点位点击：点击任一点位，弹出底部最速攻略卡片并更新内容。
- 自动推荐：进入页面后延时自动弹出推荐点位卡片。
- 底部轮播：展示扩展范围内可去地点，支持左右滑动。
- 面板互斥：`quickCard` 与 `nearbyCarousel` 互斥显示。
- 当前位置：显示“現在地”，并支持定位按钮回到定位点。
- 地图降级：JS API 不可用时自动切换 Embed/覆盖层可交互方案。
- 文案：用户可见文案统一为日文。

## 3. 关键业务规则
- 推荐点：`places` 中 `recommended=true` 的点位优先作为自动弹卡对象。
- 轮播候选：优先取“扩展范围内但当前画面外”的地点；不足时回退到扩展范围内地点。
- 轮播信息最小化：地点、距离、剩余时间、开始时间。
- 当前位置策略：测试阶段固定到隅田川/天空树附近，不读取设备真实定位。

## 4. 与产品语义一致性
- `へ` 是语义抽象，不强制写入地点展示名。
- 地点展示名使用真实地点名（活动名/地名）。
- `最速攻略` 明确为快速查看，不等于详情。

## 5. 验收要点（阶段）
- 首屏进入可见全屏地图与点位。
- 点击点位后卡片内容正确切换。
- 卡片显示时轮播隐藏，卡片关闭后轮播恢复。
- 轮播可横向滑动，能展示视图外扩范围候选。
- 右侧定位按钮可将视图回到定位点。
- 文案为日文，排版稳定。

## 6. 暂未纳入本阶段
- 真实 GPS 定位与定位权限管理。
- 详情页深层跳转。
- 后端实时活动数据接入与动态排序。
