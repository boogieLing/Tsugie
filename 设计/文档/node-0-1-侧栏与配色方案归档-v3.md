# Node 0:1 侧栏与配色方案归档 v3

## 1. 本阶段目标
- 在不破坏既有主链路（地图 -> quickCard -> 详情）的前提下，为首页加入可持续扩展的侧栏入口。
- 将“主题换色”从静态样式升级为可交互的运行时能力，支持快速视觉验证。
- 保持 iOS 高雅简约风与当前页面一致，避免新增模块成为视觉噪点。

## 2. 本阶段需求落地
1. 点击 `top-kaomoji` 后，从右侧弹出侧栏。
2. 侧栏菜单项日文本土化，当前保留：
- お気に入りスポット
- 通知設定
- お問い合わせ
3. 菜单选中态：左侧糖果色发光小胶囊 + 更强发光背景；选中态不使用边框。
4. 侧栏顶部新增配色小胶囊：
- 不显示“配色”文案，仅显示当前方案颜色。
- 点击后展开配色方案块。
5. 配色方案块：
- 不显示文案，仅显示小尺寸颜色/渐变块。
- 一键切换主题方案。
- 顶部胶囊颜色必须与当前选中方案保持一致。
6. 暂时移除“もうひとつの世界”侧栏菜单项（保留对应底层能力，不暴露入口）。

## 3. 功能点（当前可用）
### 3.1 侧栏开关与关闭路径
- 打开：点击 `top-kaomoji`。
- 关闭：点击遮罩、点击关闭按钮、按 `Esc`。
- 保护：详情页打开时不再展开侧栏；进入详情会自动收起侧栏。

### 3.2 侧栏模块行为
1. お気に入りスポット
- 展示收藏地点列表（含名称、距离、状态、时间段）。
- 点击条目直接打开该地点 quickCard。

2. 通知設定
- 两个可切换项：开始前提醒、周边候选提醒。
- 当前为前端状态管理，便于后续接入系统通知权限与持久化。

3. お問い合わせ
- 邮件联系入口。
- 支持一键复制联系邮箱（含复制成功/失败反馈）。

### 3.3 主题配色能力
- 方案：`fresh`、`ocean`、`sunset`、`sakura`、`night`。
- 主题切换后即时作用于关键视觉层：
- 顶部配色胶囊
- 配色方案选中态
- 地图 tint
- 侧栏背景渐变
- 侧栏选中态发光
- CTA 渐变色

## 4. 视觉与交互设计重点
1. 信息密度与识别效率
- 配色入口与配色块均改为“纯颜色语义”，减少文案占位，提升扫视效率。
- 在小屏下保证可点击面积与分组节奏，避免误触。

2. 选中态层级
- 未选中：半透明阴影，维持轻层次。
- 选中：增强发光背景 + 左侧糖果色发光胶囊。
- 选中态显式无边框，避免“描边 + 发光”叠加造成脏边。

3. 配色一致性
- 顶部胶囊直接使用当前方案真实渐变色，不做额外泛白混色。
- 解决“显示色与实际方案不一致”的认知偏差。

4. 风格一致性
- 新增组件沿用玻璃质感与圆角体系，避免出现独立设计语言。
- 动效以短时过渡为主，控制张力，不抢地图主内容。

## 5. 当前约束与后续接入建议
1. 约束
- 通知设置尚未接系统权限与持久化。
- 收藏夹尚未接“地图内收藏/取消收藏”入口。
- 主题方案为前端内置配置，未接服务端下发。

2. 后续建议
- 将通知设置映射到本地存储，并在首次开启时触发权限流程。
- 增加地图点位收藏动作，打通“收藏 -> 侧栏 -> quickCard”闭环。
- 将主题变量沉淀为统一 token（便于 Figma 与前端对齐）。

## 6. 验收清单
1. 侧栏基础
- `top-kaomoji` 可稳定开关侧栏。
- 遮罩、关闭按钮、Esc 均可关闭。

2. 菜单表现
- 菜单选中态发光明显。
- 选中态无边框。

3. 配色系统
- 顶部胶囊无文案、无边框。
- 配色块无文案，点击后即时生效。
- 顶部胶囊颜色与当前方案一致。

4. 主链路回归
- 地图 -> quickCard -> 详情页链路无回归。
- 侧栏引入后不遮挡关键主流程交互。
