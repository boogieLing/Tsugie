# Node 0:1 时间维度日历语义与交互 v1

## 1. 目标
- 在地图首页补充“时间维度”入口，帮助用户按日期理解近期可去的 `へ`。
- 与地图页形成互补：
- 地图主页负责位置决策（附近去哪）。
- 日历页负责时间决策（哪天去哪）。

## 2. 语义定义（必须区分）
- 地图（首页）语义：`位置维度统计与决策`。
- 日历（時めぐり）语义：`时间维度统计与决策`。
- 同一批 `へ` 数据在两个语义下分别投影：
- 地图按地理空间组织。
- 日历按日期分桶组织。

## 3. 信息结构
- 入口位置：主页右上角，按钮名 `時めぐり`。
- 页面形态：全屏日历页（不是底部卡片）。
- 每日卡片内容：
- 日期标题（如 `2/14 (土)`）。
- 右上角类别统计（含 logo 占位 + 数量）。
- 默认仅显示 2 条：距离最近且开始更早的候选。
- 点击行为：
- 点某一天空白区域：右侧侧栏弹出该天列表。
- 点当天 2 条之一：直接关闭日历并打开 quickCard。

## 4. 排序与筛选规则
- 每日默认展示条数：`2`。
- 推荐顺序（用于每日 top2 与日侧栏列表）：
1. 距离优先（当前定位到 `へ` 的距离升序）
2. 开始时间优先（开始更早者优先）
3. 热度兜底（heat 降序）
- 日侧栏分类：
- 默认分类：`全部`
- 可扩展分类：`花火 / 祭典 / 景観 / 其他`

## 5. 分类与 Logo 占位
- 分类展示位都预留 logo：
- 日历每日右上角统计条目（logo + count）。
- 日侧栏筛选 chip（logo + 名称 + count）。
- 日侧栏列表项（logo + 名称 + 信息）。
- 当前测试阶段允许使用占位字符（如 `花 / 祭 / 景 / 他`），后续替换为正式图标资源。

## 6. 面板互斥与状态机
- 日历打开时采用“独立页面切换”：
- 地图主页 UI（地图、nearby、quickCard、detail、主页侧栏）整体隐藏。
- 不修改主页组件内部状态，仅做页面级可见性切换。
- 关闭日历后恢复进入前主页状态。
- 按天打开侧栏时：
- 使用日历专属右侧抽屉承载当天分类结果（不复用主页侧栏）。
- 关闭日历抽屉不关闭日历（保持时间语义上下文）。
- `Esc` 关闭优先级：
1. 侧栏
2. 日历
3. 详情页
4. quickCard

## 7. 视觉与风格约束
- 维持现有 iOS 极简磨砂风。
- 日历页与侧栏延续同一 token：
- 淡色渐变底 + 半透明卡片 + 低对比边框。
- 字体与间距继承主页体系，避免新语义引入新视觉语言。
