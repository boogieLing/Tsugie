# つぎへ (Tsugi e)

最聪明的花火导航。

つぎへ 是一款基于时间与地理位置的移动应用，目标是在「当前时间 + 当前地点」条件下，为用户给出下一场值得去的活动。

## MVP 目标

- 以最小功能验证核心价值
- 快速上线，获取真实用户反馈
- 严格控制范围，优先可用性

MVP 核心体验：打开 App 即进入地图，自动定位并给出今日推荐，用户可查看详情并一键导航。

## 项目目录

本项目采用固定五大模块组织：

- `设计/`：Figma 产物、原型、设计文档
- `ios开发/`：iOS 客户端实现（Swift/SwiftUI）
- `数据端/`：活动数据、推荐排序、接口相关实现
- `需求/`：需求文档与版本说明
- `记录/`：项目过程记录、决策与归档（含 `记录/归档/`）

## 工作流程

1. 需求先行：先更新 `需求/`，再进入实现。
2. 设计沉淀：设计稿、交互说明统一进入 `设计/`。
3. 分端开发：
- iOS 客户端代码放 `ios开发/`
- 数据与规则实现放 `数据端/`
4. 联调验收：严格按 MVP 边界，不扩功能。
5. 过程记录：关键决策与里程碑写入 `记录/`，历史内容归档到 `记录/归档/`。

## 变更纪律

出现以下重大变化时，必须同步更新：

- `AGENTS.md`
- `记录/项目变更记录.md`
- 受影响的 `需求/` 文档

重大变化包括：

- MVP 范围变化
- 目录结构调整
- 推荐逻辑或关键业务规则变化
- 技术栈或端到端流程变化
- 里程碑计划变化
